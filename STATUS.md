# Оборудыш — статус разработки (03.07.2026)

## ✅ Готово и проверено

### Приложение (prototype/index.html)
- Все экраны v1: регистрация → мастер заявки (календарь, каталог, корзина, детали) → мои заявки → 626 → админ → старший.
- Стиль Media BMSTU: чёрный + зелёный #43ed4e, Montserrat, кнопки-пилюли; тёмная/светлая тема, тумблер в шапке (выбор запоминается).
- Каталог из парсера: `prototype/catalog.js` (11 категорий, 200 позиций) подхватывается автоматически; фото — `prototype/img/<Короткое имя>.jpg` (папку с фото положить туда же).
- Доступы без слова «пароль»: «недоступна»/«категория временно недоступна»; код вводится по двойному тапу на «+» позиции или на карточке категории.
- «Ответственный за медиа» — на первом шаге, до каталога.
- Регистрация: нельзя без ФИО; MB — минимум один отдел; СО/ССФ — минимум одна организация (и «Другое» с текстом); мультивыбор организаций; отделы MB выше списка организаций.
- Живое обновление: данные подтягиваются каждые 12 с; введённый текст и позиция скролла переживают обновление.
- Фото сдачи: 3 реальных фото (камера/галерея, сжатие на телефоне) — и для оборудования, и для 626.
- Чат в стиле TG: свои справа зелёным, собеседник слева.
- Иконки — единый SVG line-art (эмодзи убраны).
- Чек-лист выдачи: всё отмечено по умолчанию; админ может убрать позицию (нет в наличии) и добавить замену из каталога.
- Telegram: полный экран, BackButton по шагам, высота по вьюпорту (не залезает на интерфейс TG), тема из Telegram + ручной тумблер.

### Бэкенд (bot/main.py)
- API + статика + бот в одном процессе; SQLite; авторизация по подписи initData; Python 3.8+.
- Регистрация с верификацией: MB — авто, СО/ССФ — вручную старшими (роль, отклонение, чёрный список).
- Заявки: полный цикл new → куратор → согласована → выдана (с заменой состава) → возврат (фото) → закрыта/к старшим.
- 626: цикл new → согласована (старшим) → куратор → сдача (фото).
- Уведомления от бота на каждый шаг; карточка заявки в админ-канал, обновляется при смене статуса.
- Фото: пользователь сдал → фото куратору в личку; админ принял возврат → фото в канал.
- Роли из .env: ADMIN_IDS, SENIOR_ADMIN_IDS; ADMIN_CHAT_ID — канал (id канала: переслать пост боту в личку).
- DEV-режим (DEV_USER_ID) для локальных тестов без Telegram.
- Прогнан e2e-тест: 21/21 (регистрация, валидации, весь цикл заявки, замена состава, фото, 626, чат, отмена).

## Обновление на сервере
1. Заменить `prototype/index.html`, `prototype/texts.js`, `prototype/catalog.js`, `bot/main.py`, `bot/texts.py`.
2. В `bot/.env` дописать новые переменные по `bot/.env.example` (ADMIN_CHAT_ID — теперь канал).
3. Перезапустить бота. Зависимости не менялись. Папку с фото оборудования — в `prototype/img/`.
