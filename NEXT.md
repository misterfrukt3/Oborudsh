## Что сделать дома

### 1. Подключить закрытую таблицу людей

Таблица:

```text
1caWlBJbzYHt0-SK744UD9rU7VS28cNAJY7jxoXosqCs
```

Лист: `люди`.

Нужно:

1. Создать Google Service Account и дать ему доступ «Читатель».
2. Посмотреть точные заголовки столбцов ФИО, организаций, отделов и ролей.
3. Добавить настройки `MEMBERS_SHEET_ID`, `MEMBERS_SHEET_TAB` и использовать общий JSON сервисного аккаунта.
4. Переделать регистрацию:
   - сначала пользователь вводит ФИО;
   - найден в таблице — организации и отделы заполняются автоматически, назначается максимальная доступная роль, верификация проходит сразу;
   - не найден — открывается ручная форма и заявка уходит старшим;
   - несколько одинаковых ФИО — только ручная верификация.

Сейчас закрытая таблица людей ещё не подключена: старая проверка Media BMSTU работает через публичный `MB_SHEET_URL`.

### 2. Создать таблицу для баллов

Код записи баллов уже готов. Нужно создать отдельную Google-таблицу, дать Service Account право «Редактор» и заполнить:

```env
GOOGLE_SHEETS_ENABLED=1
GOOGLE_SHEET_ID=
GOOGLE_SERVICE_ACCOUNT_JSON_B64=
GOOGLE_SHEET_EVENTS_TAB=Начисления
GOOGLE_SHEET_SUMMARY_TAB=Админы
```

После запуска выполнить `/scorestatus`, затем `/scoresync`.

### 3. Поправить каталог оборудования

- Актуализировать `prototype/catalog.js`: названия, количество экземпляров и уровни доступа.
- Проверить уникальность `short`, положительное `total` и соответствие паспортов реальным экземплярам.
- При необходимости обновить изображения каталога.

## Быстрый старт на другом компьютере

```powershell
git clone https://github.com/misterfrukt3/Oborudsh.git
cd Oborudsh
git checkout main
cd bot
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe -m unittest -v test_core.py
```

Секреты и JSON-ключ сервисного аккаунта хранить только в `bot/.env`, в Git не добавлять.
