# 🧭 Гайд — куда развивать бота

Куда двигаться дальше: что добить перед продом, что улучшить, что добавить. Приоритет сверху вниз. Опирается на бэклог из `CLAUDE.md` + разбор кода.

---

## 🔴 Приоритет 1 — добить прод (без кода, но обязательно)

> [!danger] Пока это не сделано — на проде будут баги
> - **Nginx**: `client_max_body_size 32M`, иначе фото сдачи → ошибка 413. См. [[Уведомления и карточки]].
> - **`.env` на сервере**: вписать `MB_SHEET_URL`, `ORG_MEMBERS_FILE`, боевые `ADMIN_IDS`/`SENIOR_ADMIN_IDS` (личные id!), `ADMIN_CHAT_ID`. См. [[Конфиг и запуск]].
> - **`DEV_USER_ID` — убрать** с прода (иначе дыра в [[Авторизация|авторизации]]).
> - **Перенести `catalog.js`** на сервер вместе с фронтом (его читает и [[Каталог и занятость|бэкенд]]).
> - **Отозвать старый токен** бота (скомпрометирован).
> - Залить `prototype/img/*.avif` (фото позиций; фолбэк-иконка уже работает).

## 🟠 Приоритет 2 — проверить вживую

- **Реальный прогон в Telegram** заказчиком: media-group фото, эскалация проблемного возврата, непрочитанные бейджи, сводки 22:00, автоотмены. Локально без прод-токена бота не поднять (DEV шлёт no-op вместо сообщений).
- Прогнать [[Поток заявки]] целиком: new → … → closed + ветку эскалации.

## 🟢 Приоритет 3 — доработки существующего

### Иконки
Пройтись по `ICONS` / `UICONS` в [[Фронтенд — стили и токены]] — заказчик отмечал, что часть грубовата. Править по конкретным замечаниям.

### «Живость» вместо поллинга
Сейчас `startPolling` дёргает `me` каждые 12 сек ([[Фронтенд — режимы и запуск]]). Если мало — заменить на **SSE/WebSocket**:
- сервер: aiohttp умеет WebSocket (`web.WebSocketResponse`) — держать соединение, слать `boot_payload` при изменении;
- фронт: подписка вместо `setInterval`.
- Средняя по сложности задача, трогает и фронт, и бэк.

### Миграция старых данных
Перелить пользователей/заявки из sqlite старого бота (`bot_fixed.py`) в новую схему ([[Слой БД]]). Разовый скрипт: читать старую базу → `INSERT` в новую через `db()`.

## 🔵 Приоритет 4 — новые фичи (v2-хвост)

### QR-сканер выдачи/возврата
- Фронт: экран сканера (Telegram `showScanQrPopup` или `<input capture>` + JS-декодер).
- Идея: на экземпляре наклейка с QR = `short + №`. Скан на выдаче/сдаче сверяет комплект.
- Затрагивает: `SCREENS.adminReq` (выдача), `returnFlow`. Новый эндпоинт не обязателен.

### Порядочность / рейтинг пользователей
- Считать: просрочки, поломки, аккуратные возвраты → баллы.
- БД: новая таблица или колонка в `users`; не забыть [[Слой БД|_migrate]].
- Есть соседний проект рейтинга (`D:\Media BMSTU\рейтинг`) — можно связать.

### Синхронизация с Google Sheets
- Двусторонняя: заявки/занятость → в таблицу для не-пользователей Telegram.
- Читать уже умеем (`mb_members` тянет CSV). Писать — через Google API (нужен сервис-аккаунт).

## 🛠️ Рецепты типовых правок

> [!example] Добавить поле в заявку (сквозь весь стек)
> 1. [[Слой БД]]: колонка в `init_db` **+** `_migrate`.
> 2. [[API-эндпоинты]]: писать её в `api_req_create` / `api_req_update`.
> 3. [[boot_payload — сборка ответа]]: отдавать в `shape_req`.
> 4. [[Фронтенд — экраны]]: показать в `reqDetail` / `adminReq`.

> [!example] Добавить действие по заявке
> - Ветка в `api_req_action` ([[API-эндпоинты]]) → проверка прав → UPDATE → `_push_hist` → `notify` → карточка.
> - Кнопка на фронте → `srvDo("request/action", {id, action:"...", ...})`.

> [!example] Новый экран
> - `SCREENS.x = (p)=>({body, mainbtn})` + `navTo('x')`. См. [[Фронтенд — навигация и render]].

> [!example] Новое уведомление/напоминание
> - Разовое: `notify(uid, text)` в нужной ветке.
> - Периодическое: ветка в `run_checks` ([[Планировщик]]) + флаг в `notif`, чтоб не спамить.

## ⚠️ Что помнить всегда

- Правил `main.py` → **перезапуск бота**. Правил `prototype/*` → нет.
- Новая колонка → **обязательно в `_migrate`**, иначе прод-база упадёт.
- Python **3.8** на VPS — без синтаксиса 3.9+.
- Токены — только в `.env`, никогда в код/чат/коммиты.
- Строки в `onclick` на фронте — через `escJs`.
