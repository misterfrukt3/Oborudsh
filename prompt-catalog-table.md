# Промт для нейросети: таблица «файл картинки → оборудка»

Как этим пользоваться: приложение показывает фото позиции из файла `prototype/img/<Короткое имя>.avif`,
где `<Короткое имя>` — это поле `short` из `prototype/catalog.js` (точь-в-точь, с теми же пробелами/регистром).
Если файла нет — показывается SVG-иконка категории. Задача промта — сверить имена файлов картинок с каталогом
и построить таблицу соответствия (плюс подсказать, какие файлы переименовать, а каких позиций фото не хватает).

## Что дать нейросети на вход
1. Список имён файлов из папки с картинками (например: `ls prototype/img` → вставить как список).
2. Файл `prototype/catalog.js` целиком (в нём массив категорий с полями `short`, `full`, `total`, `level`).

## Готовый промт (скопировать вместе с двумя списками)

```
Ты — ассистент по каталогу съёмочного оборудования «Оборудыш».

ДАНО:
1) СПИСОК ФАЙЛОВ картинок (имя файла без расширения = название оборудки):
<сюда вставить список имён файлов, по одному в строке>

2) КАТАЛОГ — JavaScript-массив window.OBORUDKA_CATALOG из catalog.js
   (каждая позиция: {short, full, total, level?}, сгруппированы по категориям {cat, items:[...]}):
<сюда вставить содержимое catalog.js>

ЗАДАЧА:
Построй Markdown-таблицу со столбцами:
| Имя файла | short (из каталога) | Полное название (full) | Категория | Статус |

Правила:
- Для каждого имени файла найди позицию каталога, к которой оно относится. Сопоставляй по смыслу
  (модель камеры/объектива/света), даже если имя файла написано иначе (латиница/кириллица, лишние пробелы,
  другой регистр, сокращения).
- В столбец «short» подставляй ТОЧНОЕ значение поля short из каталога (это имя, под которым файл должен лежать:
  prototype/img/<short>.avif).
- Статус:
  · «OK» — имя файла уже совпадает с short буква-в-букву;
  · «ПЕРЕИМЕНОВАТЬ → <short>.avif» — файл есть, но имя не совпадает с short;
  · «НЕ ИЗ КАТАЛОГА» — файл не удалось сопоставить ни с одной позицией.
- После таблицы отдельным списком «НЕТ ФОТО» перечисли все short из каталога, для которых файла нет вообще.
- Ничего не выдумывай: если не уверен в сопоставлении — помечай «НЕ ИЗ КАТАЛОГА» и поясняй одной строкой.

ФОРМАТ ВЫВОДА: сначала таблица, затем список «НЕТ ФОТО», затем (если были переименования) готовый блок
команд для переименования в PowerShell и bash.
```

## Замечания
- Итоговые файлы должны лежать в `prototype/img/` в формате **.avif** (фронт уже ищет `.avif`).
- `short` берётся из `catalog.js` — если заказчик обновит каталог парсером, таблицу нужно пересобрать.
- Регистр и пробелы критичны: `img/Сони а7м4.avif`, а не `img/sony-a7m4.avif`.
