Ты продолжаешь работу над «Оборудышем» — Telegram Mini App бронирования съёмочного оборудования и студии 626 Media BMSTU (МГТУ им. Баумана). Этот файл — единственный источник контекста; читай его целиком перед работой.

## ⚠️ Безопасность
1. Тестовый бот — токен только в `bot/.env`. Токены не писать в код/чат/документы/коммиты.
2. `DEV_USER_ID` на проде не задавать (обходит проверку подписи).

## bot/.env
BOT_TOKEN, WEBAPP_URL, PORT=8737, ADMIN_CHAT_ID (канал), ADMIN_IDS, SENIOR_ADMIN_IDS (личные id через запятую), DEV_USER_ID (только локально), MB_SHEET_URL (CSV листа «список ребят» для автосверки MB; пусто = выкл), ORG_MEMBERS_FILE (файл со списком организаций от make_members.py; дефолт bot/org_members.csv). Тело запроса поднято до 32 МБ прямо в aiohttp (`client_max_size`), поэтому фото сдачи проходят без внешнего веб-сервера.

## Как всё устроено

### Фронтенд (prototype/index.html + prototype/style.css)
- `SCREENS` — объект экранов, `stack`+`render()` — навигация. `SRV` = ответ бэкенда (null → демо на моках). `api(path,body)` шлёт POST /api/… с `initData`; `srvDo(path,body,after)` = api+applyBoot+after; `applyBoot(d)` заливает profile/requests/bookings626/verifQueue/users/dayload/busy626/favSets.
- `render()` сохраняет скролл и введённый текст (input/textarea с id) при перерисовке того же экрана; поллинг `startPolling()` — api('me') каждые 12 с (пауза при открытом overlay/фокусе в поле, перерисовка только при изменении данных).
- В Telegram (`IN_TG`): рамка скрыта, BackButton по шагам (`tgLeft`), высота по `viewportStableHeight`, тема из Telegram + ручной тумблер (`userToggleTheme`, localStorage `obor_theme` перебивает), профиль моков очищается, ФИО — с сервера.
- Доступы: **кода/пароля БОЛЬШЕ НЕТ** (secretTap/pwModal вырезаны). `itemLocked`: `глава` → только старшие (`isSeniorNow()`), `акт` → `canAct()` (активист/production или чекбокс «ответственный за медиа» на шаге 1). Заблокированная категория просто показывается недоступной; блокировки категорий приходят с сервера (`catBlocks`) и применяются в `rebuildCatalog`.
- Каталог: `CATALOG_BASE` (из catalog.js) неизменяем; `let CATALOG` пересобирается в `applyBoot` через `rebuildCatalog(extraItems, catBlocks, removedItems)`. `escJs()` экранирует `short` во всех onclick (в каталоге есть кавычки в названиях!).
- Регистрация: ФИО — **три слова** (`regValid()`), MB → минимум один отдел, СО/ССФ → минимум одна организация; чипы роли («Твоя роль») только для MB (активист/стажёр/production) — **СО/ССФ больше не чип**, ставится бэкендом автоматически, если MB не выбран (`regPayload()` шлёт `role:""`). Перед регистрацией — экран `SCREENS.manual` (методичка), ссылка на него безусловна и с экрана `rules` (и до регистрации, и после).
- Фото сдачи: `photosGrid/addPhotos/shrinkPhoto` (сжатие до 1024px JPEG 0.65, base64) — несколько сразу (min 1, max 5) + коммент, галерея с удалением. Экраны `returnFlow` и `handover626`. Гард по размеру от 413.
- Чат: `chatMsg(m,invert)` — подпись «кто пишет» (`m.label`), красный класс `.msg.senior` (только если писали из панели старшего). Непрочитанные — `unreadBadge`/`markRead` (вызов в `after` детальных экранов), бейджи на карточках и на входах «Служебное».
- Роль-режимы: `role` (user/admin/senior), `setRole` + вход через «Служебное»; `isSeniorNow()` = `SRV?SRV.isSenior:role==='senior'`.
- Чек-лист выдачи: всё отмечено по умолчанию; ✕ убирает позицию, «➕ Добавить позицию» — из каталога; изменённый состав уходит в `issue` (`items`).
- Иконки — SVG line-art: `ICONS` (категории) + `UICONS`/`uic()` (UI). Эмодзи как структурные иконки не использовать.
- Ловушки CSS: `font:` шорткат не принимает `inherit`-семейство; mobile-overrides требуют `.phone .класс`; блок `body.in-tg` в конце `style.css`. Типографика/радиусы — через токены `--fs-*`/`--r-*` в `:root` (`style.css`), новые размеры добавлять туда, а не магическими числами по месту.

### Бэкенд (bot/main.py)
- Один процесс: aiohttp API + статика + aiogram-бот (polling) + scheduler-task. Авторизация — HMAC-подпись `initData` (декоратор `auth`), `touch_user` обновляет username/фото.
- Роли: `ADMIN_IDS`/`SENIOR_ADMIN_IDS` из .env (старший ⊇ админ). В списках — ЛИЧНЫЕ id (не чатов!), иначе уведомления о верификации летят в общий чат (есть warning в лог).
- Каждый мутирующий эндпоинт возвращает свежий `boot_payload(uid)` — фронт просто делает applyBoot.
- Статусы заявки: new→curator→approved→issued→ret→closed / rejected / canceled. `escalated=1` — проблемный возврат ушёл старшим (куратор не принимает, `return`+comment только для не-старшего). `uncurator` — снять кураторство (до закрытия). 626: new→approved→ret→closed / rejected / canceled.
- Таблицы (сверх базовых): `extra_items` (добавленная оборудка), `removed_items` (скрытая), `cat_blocks` (until — авто-снятие в scheduler), `fav_sets`, `reads` (непрочитанные: seen), `meta` (digest_date/backup_date). `messages.role` — из какой роли писали (для подписи/красного). `users.block_until` — авто-снятие блока. `_migrate()` догоняет колонки (**новую колонку/таблицу заводить и в `init_db`, и в `_migrate`**).
- Фото: **на диск НЕ пишем**. `_decode_photos` → `_send_blobs`/`send_photos_b64` (`BufferedInputFile`) шлют сразу куратору+в канал при сдаче (userret/handover).
- Уведомления: `notify()`, `notify_seniors()`, карточка в `ADMIN_CHAT_ID` (`send_or_update_card`). В карточке — пометка «поздняя выдача/возврат» (сб после 18:00, `late_note`). `api_chat`: старший уведомляет и владельца, и куратора.
- Планировщик (`run_checks`/`scheduler_loop`, 5 мин): напоминания о сдаче/просрочке; автоотмены; **напоминания в канал о зависших** (без куратора/не согласовано/626 >6ч); **ежедневная сводка 22:00** (`daily_digest`, гейт `meta`); **бэкап раз в неделю** (`weekly_backup`, храним 3); авто-снятие истёкших блоков (юзеры/категории).
- Верификация: MB — по `MB_SHEET_URL` (Google CSV, `mb_ok`), организации — по `ORG_MEMBERS_FILE` (`org_ok`, файл от `make_members.py`); перепроверка при смене имени; не нашли → pending, уведомление **в канал**. `api_verify`: ok/no(с причиной)/block(days/term→block_until)/unblock. Блокировка авто-снимается по сроку.
- Эндпоинты (сверх базовых): `/api/read`, `/api/appeal`, `/api/user/role|delete`, `/api/category/block` (блок → уведомление в `ADMIN_CHAT_ID`), `/api/equip/add|del|remove|restore`, `/api/favset/add|del`, `/api/broadcast`, `/api/stats` (+ `topRejecters`), `/api/export {requests|626|admins}` (текст через `build_export_text()`, без markdown-таблиц), `/api/dev/tick`.
- Скрытые команды бота в личке (не в меню, только старшим, остальным бот не отвечает):
  - `/digest` — прислать ежедневную сводку прямо сейчас (не ждать 22:00).
  - `/backup` — сделать бэкап базы прямо сейчас (не ждать раз в неделю).
  - `/checks` — прогнать плановые проверки прямо сейчас (напоминания о сдаче/просрочке, автоотмены, напоминания о зависших заявках).
